# ZZZQ.IT.COM - 글로벌 엔터테인먼트 플랫폼

## 📌 프로젝트 개요
# 이 프로젝트는 글로벌 사용자를 대상으로 한 엔터테인먼트 웹사이트이다.
# 다양한 참여형 컨텐츠(투표, 설문, 퀴즈, 토너먼트)를 제공하고,
# 사용자 참여 데이터를 Supabase에 적재하여 브라우저에서 차트/그래프로 시각화하는 것이 핵심 목적이다.
# 글로벌 트래픽을 유도하고 Google AdSense 수익을 창출한다.
#
# 도메인: zzzq.it.com
# 기술 스택: Next.js 15 + TypeScript + Tailwind CSS + App Router
# 데이터베이스: Supabase (PostgreSQL)
# 배포: Vercel (GitHub main 브랜치 자동 배포)
# 대상: 전 세계 사용자 (다국어 지원 필수)

---

## 🔴 필수 규칙 (반드시 지켜야 함)

### 1. 주석은 반드시 한글로 작성
# 모든 코드 주석은 한글로 작성한다.
# 변수명, 함수명, 컴포넌트명은 영문으로 작성한다.
# 예시:
#   // 사용자 투표 데이터를 서버에 전송하는 함수
#   const submitVote = async (voteData: VoteData) => { ... }

### 2. 커밋 및 배포 전 버전 확인 필수
# 코드 변경 후 커밋하기 전에 반드시 사용자에게 버전명을 확인받는다.
# 절대 임의로 커밋하거나 푸시하지 않는다.
# 확인 형식: "v{버전} - {변경사항 요약}" (예: "v0.1.0 - 메인 페이지 레이아웃 구성")
# 사용자가 승인하면 그때 커밋 + 푸시를 진행한다.
# 푸시 전 반드시 `npm run build`로 빌드 성공을 확인한다.

### 3. 코딩 후 오류 점검 필수
# 코드 작성 또는 수정 후 반드시 아래 점검을 수행한다:
#   1) npm run build — 빌드 에러 없는지 확인
#   2) TypeScript 타입 에러 없는지 확인
#   3) ESLint 경고/에러 없는지 확인
#   4) 브라우저 콘솔에 에러 없는지 확인
# 에러가 있으면 커밋하지 않고 반드시 수정 후 다시 점검한다.
# "일단 커밋하고 나중에 고치자"는 허용하지 않는다.

### 4. 다국어(i18n) 지원 필수
# 글로벌 서비스이므로 모든 UI 텍스트는 다국어 패키지를 통해 관리한다.
# next-intl 또는 동등한 i18n 라이브러리를 사용한다.
# 기본 언어: English (en)
# 지원 언어: 한국어(ko), 일본어(ja), 중국어(zh), 스페인어(es), 프랑스어(fr) 등 확장 가능하게 구성
# 하드코딩된 텍스트는 허용하지 않는다. 모든 텍스트는 번역 키를 사용한다.

### 5. 애드센스 배치 규칙
# 광고는 항상 정해진 영역에만 배치하며, 임의로 위치를 변경하지 않는다.
# 허용된 광고 위치:
#   - 헤더 하단 배너 (728x90 leaderboard)
#   - 컨텐츠 사이 (in-feed 네이티브 광고, 라운드 사이 등)
#   - 결과 페이지 (300x250 또는 336x280 디스플레이)
#   - 사이드바 (160x600, PC 전용 / 모바일에서는 컨텐츠 하단으로 이동)
# 금지 사항:
#   - 투표 버튼, 참여 버튼 근처에 광고 배치 금지 (오클릭 유발 → 정책 위반)
#   - 팝업, 오버레이 형태의 광고 금지
#   - 한 페이지에 광고 4개 초과 금지
# 광고 슬롯은 공통 컴포넌트로 관리한다: <AdSlot position="header|between|result|sidebar" />

### 6. 퍼블릭 데이터 모델 (로그인 없음)
# 이 서비스는 로그인/회원가입 기능이 없다. 인증 시스템을 만들지 않는다.
# 모든 데이터는 익명 퍼블릭 데이터이다.
# 테이블 설계 시 user_id, auth 관련 컬럼을 만들지 않는다.
# 데이터 식별은 IP 기반 중복 방지 또는 브라우저 fingerprint 등 경량 방식만 사용한다.
# Supabase RLS 정책:
#   - SELECT: 누구나 읽기 가능 (anon 허용)
#   - INSERT: 누구나 쓰기 가능 (anon 허용)
#   - UPDATE/DELETE: 필요 시 서비스 관리자만 (service_role 키 사용, 프론트에서 직접 불가)
# Supabase Auth 모듈은 사용하지 않는다.
# 절대 로그인 페이지, 회원가입 폼, OAuth 연동 등을 만들지 않는다.

### 7. 데이터는 Supabase에 적재, 브라우저에서 시각화
# 모든 사용자 참여 데이터(투표, 설문, 퀴즈 등)는 Supabase(PostgreSQL)에 저장한다.
# 프론트엔드(브라우저)에서 Supabase 클라이언트로 데이터를 조회하여 차트/그래프로 보여준다.
# 서버 사이드 렌더링(SSR)과 클라이언트 사이드 조회를 적절히 조합한다.
# 환경변수:
#   - NEXT_PUBLIC_SUPABASE_URL: Supabase 프로젝트 URL
#   - NEXT_PUBLIC_SUPABASE_ANON_KEY: Supabase 공개 키
# Supabase 클라이언트는 src/lib/supabase.ts에서 싱글톤으로 관리한다.

### 8. 모바일 최적화 필수
# 이 웹 서비스는 모바일 환경에서도 완벽하게 동작해야 한다.
# 모든 페이지와 컴포넌트는 반드시 모바일 퍼스트(mobile-first)로 설계한다.
# 터치 인터랙션을 고려한 UI (버튼 크기 최소 44px, 충분한 간격)
# 모바일 뷰포트에서 가로 스크롤이 발생하지 않도록 한다.
# 차트/그래프도 모바일 화면에 맞게 반응형으로 렌더링한다.
# 테스트 시 반드시 모바일 뷰(375px, 390px, 414px)에서 확인한다.
# 사이드바 광고는 모바일에서 컨텐츠 하단으로 자동 이동한다.

## 🏗️ 프로젝트 구조

### 디렉토리 규칙
# app/                    → 페이지 라우트 (App Router)
# app/[locale]/            → 다국어 라우팅 루트
# src/components/          → 재사용 가능한 UI 컴포넌트
# src/components/ads/      → 애드센스 관련 컴포넌트
# src/components/charts/   → 차트/그래프 컴포넌트
# src/components/contents/ → 컨텐츠별 컴포넌트 (투표, 퀴즈, 토너먼트 등)
# src/components/layout/   → 레이아웃 공통 컴포넌트 (헤더, 푸터, 사이드바)
# src/lib/                 → 유틸리티 함수, API 클라이언트, Supabase 클라이언트
# src/types/               → TypeScript 타입 정의
# src/data/                → 컨텐츠 데이터 (투표 항목, 퀴즈 문제 등)
# messages/                → 다국어 번역 파일 (en.json, ko.json, ja.json 등)
# public/                  → 정적 파일 (이미지, 아이콘 등)

## 📊 핵심 기능 흐름 (컨텐츠 → Supabase 적재 → 브라우저 시각화)

### 컨텐츠 제작
# 다양한 참여형 컨텐츠를 만든다:
#   - This vs That 토너먼트 (이미지 기반 선호도 대결)
#   - 글로벌 설문조사 (국가별 응답 수집)
#   - 랜덤 대결 (랜덤 2개 항목 투표)
#   - 나라 맞추기 퀴즈 (국기/음식/랜드마크)

### 데이터 적재 (Supabase)
# 모든 사용자 참여 데이터를 Supabase에 익명으로 저장한다:
#   - 투표 결과 (선택 항목, 국가, 타임스탬프)
#   - 설문 응답 (답변, 국가, 카테고리)
#   - 퀴즈 점수 (점수, 국가, 소요시간)
# user_id 없이 익명 데이터로 적재한다.
# Supabase 테이블 설계 시 인덱스와 RLS 정책을 함께 설정한다.

### 데이터 시각화 (브라우저)
# Supabase에서 조회한 데이터를 브라우저에서 차트/그래프로 보여준다:
#   - 실시간 투표 결과 바 차트
#   - 국가별 참여 세계지도 (World Map)
#   - 시간대별 트렌드 라인 차트
#   - 카테고리별 파이 차트
#   - 글로벌 랭킹 리더보드
# 차트 라이브러리: recharts 또는 Chart.js 사용
# 모든 차트는 모바일에서도 읽기 쉽게 반응형으로 구현한다.
